# 대회 분류 체계 초안

> Korea Open 2026 (KRSA 주관) 기준. 추가 정보 파악 시 보완 예정.

---

## 1. 분류 축 (3개)

| 축 | 설명 | 예시 |
|---|---|---|
| **종별 (Division)** | 연령/레벨 구분 | 초등부, 중등부, 고등부, 대학부, 일반부 |
| **종목 (Event Type)** | 경기 유형 | 30초 번갈아뛰기, 이중뛰기, 4인 릴레이, 개인 프리스타일 등 |
| **라운드 (Round)** | 진행 단계 | 예선, 결선 (단판제 대회는 "본선" 하나) |

## 4. DB 구조 변경 제안

### 현재 구조의 한계

```
Participant.category = "개인줄넘기-초등"  ← 종별+종목이 한 필드에 혼합
Heat.heat_number = 3                     ← 코트 내 순번일 뿐, 어떤 종목인지 모름
```

### 제안 A: 프로그램 테이블 신설 (추천)

대회 당일 "진행표" 를 그대로 DB 에 넣는 구조.

**새 테이블: `programs` (경기 프로그램)**

| 필드 | 타입 | 설명 | 예시 |
|---|---|---|---|
| id | UUID | PK | |
| event_id | UUID | FK → events | |
| order | int | 진행 순번 | 1, 2, 3... |
| division | varchar(50) | 종별 | "초등 고학년" |
| event_type | varchar(50) | 종목 | "30초 번갈아뛰기" |
| event_code | varchar(30) | 종목 코드 | "sr_speed_30" |
| round | varchar(20) | 라운드 | "예선", "결선" |
| heat_duration_sec | int | 히트 시간 (초) | 30 |
| participants_per_heat | int | 히트당 인원 | 1 |
| scheduled_time | time | 예정 시각 (참고용) | "09:30" |
| note | text | 비고 | "스테이션 1~6 동시" |

**Heat 테이블 변경: `program_id` 추가**

| 필드 | 타입 | 설명 |
|---|---|---|
| program_id | UUID (nullable) | FK → programs. 이 히트가 어떤 프로그램 항목인지 |

**Participant 테이블 변경: `division` 분리**

| 필드 | 타입 | 설명 |
|---|---|---|
| division | varchar(50) | 종별 (기존 category 에서 분리) |
| category | varchar(100) | 종목 또는 세부 분류 (기존 유지, 호환성) |

### 제안 B: 간단 버전 (테이블 추가 없이)

Heat 에 직접 분류 필드 추가:

| 필드 | 타입 | 설명 |
|---|---|---|
| division | varchar(50) | 종별 |
| event_type | varchar(50) | 종목 |
| round | varchar(20) | 라운드 |

프로그램 테이블 없이 히트에 직접 태깅. 히트 생성 시 드롭다운으로 선택.

## 5. 진행 흐름 (제안 A 기준)

### 사전 준비 (대회 전날~당일 아침)

```
1. 이벤트 생성 (대회명, 날짜, 코트 수)
2. 참가자 CSV 임포트 (이름, 연락처, 소속, 종별)
3. 프로그램 등록:
   순번 1: 초등 저학년 / 30초 번갈아뛰기 / 예선
   순번 2: 초등 고학년 / 30초 번갈아뛰기 / 예선
   순번 3: 중등부 / 30초 번갈아뛰기 / 예선
   ...
4. 코트별 OBS 설정 + 연결 확인
```

### 대회 진행 중

```
1. 운영자가 현재 진행 중인 프로그램 선택 (드롭다운)
2. 히트 시작 → 자동으로 해당 프로그램의 종별/종목 태깅
3. 히트 종료 → 타임코드 기록 + 클립 자르기 대기
4. 프로그램 순번 넘기기 (다음 종목으로)
```

### 대회 후

```
1. 클립 페이지에서 종별/종목 필터링
2. 전체 자르기 또는 선택 자르기
3. 핵심 선수 클립 → Google Drive 업로드
4. 나머지 → YouTube VOD 링크 + SMS
```

## 7. 열린 질문

1. **프로그램 순서**: 실제 진행표 양식이 있는지? Excel/PDF 등 어떤 형태?
2. **종별 구분**: 초등 저/고 분리하는지 통합인지?
3. **프로그램 전환 타이밍**: 운영자가 수동으로 "다음 종목" 누르는지, 아니면 히트 생성 시 직접 선택하는지?
4. **코트별 종목 고정**: 한 코트에서 하루 내내 같은 종목만 하는지, 종목이 바뀌는지?
5. **참가자 배정 방식**: 히트에 참가자를 어떤 시점에 배정? (사전 스케줄 vs 현장 즉석)
6. **핵심 선수 마킹**: 클립 전달 대상 선수를 어떻게 구분? (참가자 필드에 플래그? 별도 리스트?)
